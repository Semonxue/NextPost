# V0.5.0 多平台支持 — 发布计划

> **状态**: 计划中  
> **目标版本**: v0.5.0  
> **基础版本**: v0.4.8  
> **创建日期**: 2026-06-04  
> **目标交付**: 新增 `title` 字段 + 支持小红书 / Instagram 多平台 + MCP 适配 opencli 平台差异

---

## 1. 背景与目标

### 1.1 现状
NextPost 当前版本 v0.4.8 主要面向 **Twitter** 单一平台。虽然 `Platform` / `PlatformConfig` 数据模型和 `DEFAULT_PLATFORM_CONFIG` 常量已经预留了多平台扩展空间，但实际端到端链路（数据 → API → UI → MCP → 外部发布）仍多处硬编码 "Twitter"。

### 1.2 目标
完成 **v0.5.0 多平台支持** 发布，让用户能够：
1. 在账号管理页**添加小红书 / Instagram / LinkedIn / Facebook 等平台的账号**
2. 创建 Post 时**携带可选的 `title` 字段**（小红书等平台必需）
3. 通过 **MCP 协议**与外部 AI 客户端对接，AI 客户端可拉取任意平台账号的待发布帖子
4. AI 客户端基于 `opencli` 完成实际发布后回传结果

### 1.3 非目标（v0.5.0 不做）
- 视频平台（TikTok / 抖音 / 视频号）原生支持
- `hashtags` / `privacy` / `coverImageUrl` / `videoMetadata` 等字段
- 多平台并行发布（一稿多发）
- PlatformPublisher 插件化重构
- PostPublishTarget 模型重构

---

## 2. 范围

本版本拆分为三个优先级：

| 级别 | 范围 | 文件数 | 是否阻塞 |
|---|---|---|---|
| **P0** | title 字段 + 多平台端到端跑通（数据/后端/MCP/UI） | ~13 | **必做** |
| **P1** | 字数限制按平台走（不做的后果：小红书写 500 字直接被前端拦截"超 280"） | 1-2 | **强烈建议** |
| **P2** | MCP 适配 opencli 平台差异（extractedTopics + 客户端对接文档） | 1 + 2 文档 | 必做（否则 AI 客户端无法发 XHS/ins） |

**预计代码量**: 15-17 个文件改动 + 2-3 个新文档  
**预计 commit 数**: 3-4 个

## 3. P0 详细任务清单

### 3.1 数据 / Schema 层（2 任务）

#### P0-1. `prisma/seed.ts` — 添加 Xiaohongshu 平台

**改动**：
- 在 platforms 数组中追加 `{ name: "Xiaohongshu", icon: "xiaohongshu" }`
- 在 configs 数组中追加 `{ name: "Xiaohongshu", maxContentLength: 1000, maxImages: 18, maxVideos: 1, allowMixedMedia: false }`

**约束**：
- `allowMixedMedia: false`（小红书图文 / 视频是分开发布的）
- 已有平台配置不动

#### P0-2. `prisma/schema.prisma` — Post 添加 `title` 字段

**改动**：
```prisma
model Post {
  // ... 现有字段保持不变 ...
  title  String?     // 可选标题（小红书必需，其他平台可选）
  // ...
}
```

**Migration**：`prisma migrate dev --name add_post_title`  
**风险**：低（nullable + 旧数据自动 NULL）

### 3.2 后端逻辑层（3 任务）

#### P0-3. `src/app/api/auth/register/route.ts` — 循环注册所有平台

**改动**：把 `prisma.platform.upsert({ where: { name: "Twitter" } })` 替换为基于 seed 列表的循环

**注意**：保持 `idempotent`（upsert 语义），不影响已有用户

#### P0-4. `src/app/api/accounts/route.ts` — POST 接 `platformId`

**改动**：
- 请求体增加 `platformId: string`（必填）
- 改 `findUnique({ where: { name: "Twitter" } })` 为 `findUnique({ where: { id: platformId } })`
- 校验 platformId 存在性（不存在返回 400）

#### P0-5. `src/app/api/platforms/route.ts` — 新建 GET 列表

**新建文件**，返回所有平台（供前端账号下拉用）：

```typescript
// GET /api/platforms
// Response: { platforms: [{ id, name, icon, maxContentLength, maxImages, ... }, ...] }
```

### 3.3 Posts API 层（2 任务）

#### P0-6. `src/app/api/posts/route.ts` — POST 接 `title`

**改动**：`createPost` 处理函数增加 `title?: string` 参数，传入 Prisma `data.title`

#### P0-7. `src/app/api/posts/[id]/route.ts` — PATCH/GET 接 `title`

**改动**：
- GET 响应回传 `title`
- PATCH 处理函数增加 `title?: string`，白名单内允许更新（仅 draft / scheduled 状态）

### 3.4 MCP 工具层（1 任务）

#### P0-8. `src/mcp/external/tools.ts` — 工具白名单 + 描述去 x.com 化

**改动**：
1. `create_post` inputSchema 增加 `title?: string`；`createPost` 实现接受 `title` 并写入
2. `update_post` inputSchema 增加 `title?: string`；`updatePost` 实现白名单允许更新
3. `report_publish_result` description 中：
   - 移除 `（如 https://x.com/user/status/123）` 平台特定例子
   - 改为通用 "（如 https://<平台域名>/<路径>/<id>）"
   - `externalPostId` 描述里 `（如 Twitter tweet ID）` 改为 `（如平台帖子 ID）`

### 3.5 UI 层（5 任务）

#### P0-9. `src/app/(main)/accounts/page.tsx` — 平台下拉 + handle 通用说明

**改动**：
- 在 `formData` state 增加 `platformId: string`
- 弹窗表单增加平台下拉框（数据来自 `GET /api/platforms`）
- label `"Twitter Handle"` → `"账号 ID"`，placeholder 改为通用 `"例如：username / 用户ID / 小红书号"`
- POST 时 body 增加 `platformId`

#### P0-10. `src/app/(main)/posts/new/page.tsx` — form 加 `title` 输入

**改动**：
- `formData` state 增加 `title: string`
- 在 content 输入框上方加 `title` 输入框（可选）
- 提交时携带 `title`

#### P0-11. `src/app/(main)/posts/[id]/edit/page.tsx` — form 加 `title` 输入

**改动**：
- `formData` state 增加 `title: string`（从后端 GET 拉取填充）
- 在 content 输入框上方加 `title` 输入框
- 提交时携带 `title`

#### P0-12. `src/app/(main)/trash/page.tsx` — 兜底文字

**改动**：`|| "Twitter"` → `account.platform?.name || "—"`（2 处）

#### P0-13. `src/app/(main)/posts/page.tsx` — 拉 `/api/platforms` 列表

**改动**：
- 已有 `platforms` state 但未拉取数据
- 在 `fetchData` 中增加 `fetch("/api/platforms")` 调用
- 平台筛选下拉框数据自动填充

## 4. P1 详细任务清单

### P1-1. `src/lib/platform.ts` — 字数限制按平台走

**改动**：
1. `DEFAULT_PLATFORM_CONFIG` 增加 `Xiaohongshu` 块：
   ```typescript
   Xiaohongshu: {
     maxContentLength: 1000,
     maxImages: 18,
     maxVideos: 1,
     allowMixedMedia: false,
   }
   ```
2. 把 `calculateContentLength` 拆为 `countCharsFor(platformName, content)`：
   - Twitter：URL 固定 23 字符的旧逻辑
   - 其他平台：直接 `[...content].length`（UTF-16 code units）
3. `getContentStatus` / `getRemainingChars` 改为调用 `countCharsFor`

**验收**：
- 单元测试：Twitter 280 字、Xiaohongshu 1000 字、Instagram 2200 字边界值
- UI 实测：小红书写 500 字不再误报"超 280"

## 5. P2 详细任务清单（MCP 适配 opencli 平台差异）

### P2-1. `src/mcp/external/tools.ts` — `extractedTopics` 计算字段

**改动**：`getPendingPosts` / `getPostDetail` 返回的 Post 对象增加 computed 字段：

```typescript
// 不入库，getPendingPosts / getPostDetail 返回时计算
extractedTopics: string[]   // 从 content 提取的 hashtag
```

**正则**：`/#([\p{L}\p{N}_]+)/gu`（中文 / 英文 / 数字 / 下划线）

**示例**：
```json
{
  "content": "今天去了 #三里屯 #brunch 真的绝",
  "extractedTopics": ["三里屯", "brunch"]
}
```

### P2-2. `docs/MCP_CLIENT_GUIDE.md`（新建）— AI 客户端对接指南

**内容大纲**：
1. 平台命令对照表：
   ```
   Twitter      → opencli twitter post <text> --images ...
   Xiaohongshu  → opencli xiaohongshu publish <content>
                   --title T --images ... --topics {extractedTopics.join(',')}
   Instagram    → opencli instagram post <caption> --media ...
                   (视频→reel，文字→note，story→story)
   ```
2. URL 模板说明：每个平台成功发布后 URL 模板
3. 平台特定字段说明（小红书 title 必填、ins 媒体类型决定命令等）
4. 错误码对照

### P2-3. `docs/PLATFORM_CONFIG.md`（新建）— 平台规则总表

**内容**：
- 各平台字数 / 图片 / 视频上限
- 字符计数算法
- 媒体类型限制
- 发布后是否返回 url（小红书不返回，twitter/ins 返回）
- 已开放 opencli 平台 vs 暂未开放平台

## 7. 风险与缓解

| 风险 | 等级 | 缓解 |
|---|---|---|
| 旧用户 Platform 表只有 Twitter | 中 | register 路由循环 upsert，触发时自动补齐 |
| Prisma migration 失败 | 低 | 本地先 `prisma migrate dev` 验证 |
| MCP 工具白名单扩字段破坏外部客户端 | 低 | title 是可选，向后兼容 |
| 小红书 publish UI automation 不稳定 | 中 | 文档明确这是 opencli 行为，NextPost 只负责规划 |
| UI 改动引入 e2e 回归 | 中 | 改动后跑 `pnpm test:e2e` |
